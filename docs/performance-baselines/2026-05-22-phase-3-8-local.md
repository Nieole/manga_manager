## 后端 Benchmark

命令：

```powershell
$env:GOCACHE='C:\Users\saber\dev\manga_manager\.gocache'
$env:GOTMPDIR='C:\Users\saber\dev\manga_manager\.tmp'
go test ./internal/database -run '^$' -bench 'BenchmarkSearchSeriesPaged_10k' -benchmem -count=1
```

结果：

```text
BenchmarkSearchSeriesPaged_10k-32    964    2106098 ns/op    129612 B/op    2514 allocs/op
```

结论：

- 1 万系列默认列表 benchmark 约 2.11 ms/op，低于阶段 3 默认列表 P95 < 300ms 的目标量级。
- 该 benchmark 是 Go 层本地基准，不包含真实浏览器网络与渲染耗时；前端渲染耗时由 Dashboard 前端观测继续记录。

## 样本生成

```powershell
go run ./cmd/sampledata -db .tmp\sample-10k.db -libraries 1 -series 10000 -books 2
go run ./cmd/sampledata -db .tmp\sample-100k.db -libraries 1 -series 100000 -books 1
```

生成结果：


## 查询计划

命令：

```powershell
go run ./cmd/queryplan -db .tmp\sample-10k.db -library 1 -series 1 -strict
go run ./cmd/queryplan -db .tmp\sample-100k.db -library 1 -series 1 -strict
```

结果：

| 样本 | 结果 | 说明 |
| --- | --- | --- |
| 10,000 系列 / 20,000 书籍 | 通过 | 首页排序、系列详情、最近阅读、Dashboard stats 和库大小统计均命中预期索引入口。 |
| 100,000 系列 / 100,000 书籍 | 通过 | 同一套查询计划在 10 万系列样本下仍满足 strict 检查。 |

关键索引入口：

| Query Case | Expected Index | Result |
| --- | --- | --- |
| home/default-updated | `idx_series_library_updated_name` | ok |
| home/name | `idx_series_library_name` | ok |
| home/created | `idx_series_library_created_name` | ok |
| home/favorite | `idx_series_library_favorite` | ok |
| series-detail/books | `idx_books_series_sort` | ok |
| recent-read/all | `idx_series_stats_last_read` | ok |
| dashboard/stats-counts | `idx_books_read_progress_series` / `idx_reading_activity_date` | ok |
| dashboard/library-sizes | `idx_books_library_size` | ok |

## 本批补强

- `cmd/sampledata` 生成样本库后再次执行迁移，确保 `series_stats` 等派生读模型已回填。
- `recent-read/all` 显式从 `idx_series_stats_last_read` 进入最近阅读读模型。
- Dashboard 库大小统计改为先按 `books(library_id, size)` 聚合，再与 `libraries` 关联。
- 新增 `idx_books_library_size` 覆盖库大小统计。
